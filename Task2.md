# 2장 실습과제

## 1. 자율주행과 로봇 분야에서 OpenCV가 사용되는 사례

OpenCV는 영상처리와 컴퓨터 비전 기능을 제공하는 오픈소스 라이브러리로, 카메라로부터 입력된 이미지나 영상에서 필요한 정보를 추출하고 분석하는 데 사용된다. 자율주행과 로봇 분야에서는 카메라 영상을 기반으로 주변 환경을 인식하고, 물체를 감지하거나 위치를 판단하는 과정에 OpenCV가 활용된다.

### 1.1 자율주행 분야에서의 활용

자율주행 자동차는 차량에 장착된 카메라, 라이다, 레이더 등의 센서를 이용하여 주변 환경을 인식한다. 이 중 카메라 영상은 차선, 신호등, 표지판, 보행자, 차량 등을 확인하는 데 매우 중요하다. OpenCV는 이러한 영상 데이터를 처리하고 분석하는 데 사용될 수 있다.

대표적인 활용 사례는 다음과 같다.

* **차선 인식**
  도로 영상에서 흰색 또는 노란색 차선을 검출하고, 차량이 차선을 따라 주행할 수 있도록 돕는다. OpenCV의 색상 필터링, 엣지 검출, 허프 변환 등의 기능이 사용될 수 있다.

* **신호등 및 표지판 인식**
  카메라 영상에서 특정 색상이나 형태를 가진 신호등과 교통 표지판을 검출한다. 이를 통해 차량이 정지, 감속, 방향 전환 등의 판단을 할 수 있다.

* **보행자 및 장애물 감지**
  영상 속에서 사람, 차량, 자전거, 장애물 등을 검출하여 충돌을 방지하는 데 활용된다. OpenCV는 객체 검출 알고리즘이나 딥러닝 모델과 함께 사용할 수 있다.

* **주차 보조 시스템**
  후방 카메라 영상에서 주차선이나 주변 장애물을 인식하여 운전자가 안전하게 주차할 수 있도록 돕는다.

### 1.2 로봇 분야에서의 활용

로봇 분야에서도 OpenCV는 카메라를 통해 주변 환경을 인식하고, 물체의 위치나 형태를 파악하는 데 사용된다. 특히 이동 로봇, 산업용 로봇, 서비스 로봇 등에서 영상처리는 중요한 역할을 한다.

대표적인 활용 사례는 다음과 같다.

* **물체 인식 및 추적**
  로봇이 카메라를 이용하여 특정 물체를 찾고, 그 물체의 움직임을 따라가도록 할 수 있다.

* **로봇 팔의 작업 보조**
  산업용 로봇 팔이 카메라로 물체의 위치를 파악하고, 정확한 위치에서 집기, 옮기기, 조립하기 등의 작업을 수행할 수 있다.

* **이동 로봇의 경로 인식**
  로봇이 바닥의 선, 마커, 장애물 등을 인식하여 이동 경로를 결정하는 데 사용된다.

* **얼굴 인식 및 사람 감지**
  서비스 로봇이나 안내 로봇은 사람의 얼굴이나 움직임을 인식하여 사용자와 상호작용할 수 있다.

따라서 OpenCV는 자율주행과 로봇 분야에서 카메라 영상으로부터 의미 있는 정보를 추출하고, 이를 기반으로 판단과 제어를 수행하는 데 중요한 역할을 한다.

## 2. CLI 개발환경이 GUI 환경보다 가지는 장점

CLI(Command Line Interface)는 명령어를 직접 입력하여 프로그램을 실행하거나 개발 환경을 제어하는 방식이다. 반면 GUI(Graphical User Interface)는 마우스와 버튼, 창과 같은 시각적 요소를 이용하여 작업을 수행하는 방식이다.

GUI는 초보자가 사용하기 쉽고 직관적이라는 장점이 있지만, 개발 환경에서는 CLI가 더 효율적인 경우가 많다.

### 2.1 빠른 작업 수행

CLI는 명령어를 직접 입력하여 파일 생성, 프로그램 실행, 폴더 이동, 패키지 설치 등을 빠르게 수행할 수 있다. GUI에서는 여러 번 클릭해야 하는 작업도 CLI에서는 한 줄의 명령어로 처리할 수 있다.

예를 들어 파이썬 파일을 실행할 때 CLI에서는 다음과 같이 간단히 입력할 수 있다.

```bash
python main.py
```

### 2.2 자동화에 유리함

CLI는 명령어를 스크립트로 작성하여 반복 작업을 자동화할 수 있다. 예를 들어 여러 파일을 한 번에 처리하거나, 프로그램 실행 과정을 자동으로 수행하도록 만들 수 있다. 이러한 자동화는 개발, 테스트, 배포 과정에서 매우 유용하다.

### 2.3 시스템 자원을 적게 사용함

GUI는 화면 구성 요소와 그래픽 처리를 위해 비교적 많은 시스템 자원을 사용한다. 반면 CLI는 텍스트 기반으로 동작하기 때문에 메모리와 CPU 사용량이 적다. 따라서 서버, 임베디드 시스템, 저사양 환경에서는 CLI가 더 적합한 경우가 많다.

### 2.4 원격 개발에 적합함

서버나 라즈베리파이 같은 장치를 원격으로 제어할 때 CLI는 매우 유용하다. SSH와 같은 원격 접속 방식을 사용하면 GUI 없이도 명령어만으로 파일 관리, 프로그램 실행, 시스템 설정 등을 수행할 수 있다.

### 2.5 개발 도구와의 연동이 좋음

CLI는 Git, Python, GCC, pip, conda 등 다양한 개발 도구와 잘 연동된다. 개발자는 명령어를 통해 코드 실행, 패키지 설치, 버전 관리, 빌드 작업 등을 효율적으로 처리할 수 있다.

따라서 CLI 개발환경은 GUI 환경보다 처음에는 어렵게 느껴질 수 있지만, 익숙해지면 작업 속도, 자동화, 원격 제어, 시스템 자원 관리 측면에서 큰 장점을 가진다.

## 3. NumPy 라이브러리가 일반 파이썬보다 빠른 이유

NumPy는 파이썬에서 배열과 행렬 연산을 효율적으로 처리하기 위해 사용되는 라이브러리이다. 일반 파이썬의 리스트도 여러 데이터를 저장할 수 있지만, 대량의 수치 계산에서는 NumPy 배열이 훨씬 빠르게 동작한다.

NumPy가 일반 파이썬보다 빠른 이유는 다음과 같다.

### 3.1 내부가 C 언어로 구현되어 있음

파이썬은 사용하기 쉽지만 실행 속도는 C 언어보다 느린 편이다. NumPy는 핵심 연산 부분이 C 언어로 구현되어 있기 때문에, 파이썬 코드로 반복문을 직접 실행하는 것보다 훨씬 빠르게 계산할 수 있다.

예를 들어 파이썬 리스트의 모든 원소에 2를 곱하려면 반복문을 사용해야 한다.

```python
arr = [1, 2, 3, 4, 5]

for i in range(len(arr)):
    arr[i] = arr[i] * 2
```

반면 NumPy는 배열 전체에 한 번에 연산을 적용할 수 있다.

```python
import numpy as np

arr = np.array([1, 2, 3, 4, 5])
arr = arr * 2
```

이처럼 NumPy는 반복문을 파이썬 코드에서 직접 처리하지 않고, 내부의 빠른 C 기반 연산으로 처리한다.

### 3.2 벡터화 연산을 지원함

NumPy는 배열 전체에 대한 연산을 한 번에 수행하는 벡터화 연산을 지원한다. 벡터화 연산을 사용하면 파이썬 반복문을 줄일 수 있고, 많은 데이터를 효율적으로 계산할 수 있다.

예를 들어 배열의 모든 요소에 같은 연산을 적용할 때 NumPy는 내부적으로 최적화된 방식으로 한 번에 처리한다. 이 때문에 이미지 처리, 행렬 계산, 과학 계산, 인공지능 데이터 처리 등에서 빠른 성능을 낼 수 있다.

### 3.3 메모리를 연속적으로 사용함

파이썬 리스트는 여러 자료형을 함께 저장할 수 있어서 유연하지만, 각 요소가 메모리상에 흩어져 저장될 수 있다. 반면 NumPy 배열은 같은 자료형의 데이터를 연속된 메모리 공간에 저장한다.

이러한 구조는 CPU가 데이터를 더 빠르게 읽고 처리할 수 있게 해준다. 특히 큰 배열이나 행렬을 다룰 때 연속적인 메모리 구조는 성능 향상에 큰 도움이 된다.

### 3.4 자료형이 고정되어 있음

파이썬 리스트는 정수, 실수, 문자열 등 서로 다른 자료형을 함께 저장할 수 있다. 하지만 NumPy 배열은 일반적으로 하나의 배열 안에 같은 자료형의 데이터가 저장된다.

자료형이 고정되어 있으면 컴퓨터가 각 데이터의 크기와 처리 방법을 미리 알 수 있기 때문에 연산을 더 빠르게 수행할 수 있다.

### 3.5 반복문 오버헤드가 줄어듦

일반 파이썬에서 반복문을 사용하면 각 반복마다 파이썬 인터프리터가 명령을 해석하고 실행해야 한다. 이 과정에서 시간이 많이 소모된다. NumPy는 반복문을 파이썬 레벨에서 직접 수행하지 않고 내부적으로 처리하기 때문에 반복문 오버헤드가 줄어든다.

결론적으로 NumPy는 C 언어 기반의 빠른 내부 구현, 벡터화 연산, 연속적인 메모리 구조, 고정된 자료형 덕분에 일반 파이썬 리스트보다 빠르게 동작한다. 따라서 대량의 수치 계산이나 영상처리, 인공지능, 데이터 분석 분야에서는 NumPy를 사용하는 것이 효율적이다.
